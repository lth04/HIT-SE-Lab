## 一、实验目的

1. 熟悉面向对象思维下的数据结构设计，掌握词语间有向图表达与操作。  
2. 用大模型辅助编程（可选）或通过算法实现图分析功能，并掌握 Git 分支/提交/推送流程。  
3. 理解功能驱动开发流程，确保 CLI 入口将输入输出集中处理，其余函数保持逻辑纯粹。

## 二、程序结构与功能

### 2.1 数据预处理与图构建

- 通过 `Lab1.processFile` 读取用户指定的文本文件（命令行参数或交互输入）。  
- 利用正则将非字母字符全部替换为空格，仅保留 a-z/A-Z，统一转小写；将空格当作分隔符生成单词序列。  
- 在 `Graph.addEdge` 中维护邻接表：`Map<String, Map<String,Integer>>`，每遇到相邻的两个单词就增加边权重；源/目标节点按需加入图中。  
- 清晰地将词列表与图结构分开，方便后续分析模块复用。

### 2.2 核心功能函数

- `showDirectedGraph(Graph G)`：打印所有节点的出边与权重，用 `node -> target(weight)` 格式在 CLI 输出；格式化到 `Graph.toString` 中，支持无出边提示。  
- `queryBridgeWords(String word1, String word2)`：检查两个词是否存在于图中，遍历 `word1` 的出边寻找直接通向 `word2` 的词，并按字典序返回一条或多条桥接词提示（无词、缺失、唯一、多个的不同提示）。  
- `generateNewText(String inputText)`：按输入文本再做一次非字母清洗，并对相邻单词查找桥接词；若存在多个，随机选一个插入；保持原单词顺序，并转回原结构。  
- `calcShortestPath(String word1, String word2)`：调用内部的 Dijkstra 实现 `calcSingleShortestPath`，使用优先队列和前驱映射找到最短路径及权值总和；支持只输入起点时输出其到其他所有词的最短路径。  
- `calPageRank(String word)`：固定阻尼因子 `d=0.85`，迭代 100 轮；对于出度为 0 的节点，将其 PR 均分给全局节点。返回指定词的 PR 值。  
- `randomWalk()`：从随机节点出发采样路径，记录走过的边；若遇到重复边或无出边即停止；将路径写入 `random_walk.txt`，并返回字符串结果。  
- 所有业务函数都不做 I/O 以保持复用性，唯一的输入与输出由 `main` 控制。

### 2.3 控制流

- `main(String[] args)`：初始通过参数或交互设定文件路径，调用 `processFile` 构建图。  
- 进入循环展示菜单（展示图、桥接词、新文本、最短路径、PageRank、随机游走、退出），按用户选项调用相应函数并打印结果；避免重复 Scanner 读取。  
- 通过 `switch` 分支封装每项功能并对非法输入给出提示。

## 三、测试与样例

### 3.1 语料

- 使用 `data/Easy Test.txt` 验证图展示与基本边权。  
- 使用 `data/Cursed Be The Treasure.txt` 做大规模的桥接词查询、生成新文本（组合短语）、最短路径（如 `to -> and`）、PageRank、随机游走等；覆盖空边、重复词、多桥等场景。

### 3.2 命令

```bash
javac -encoding UTF-8 -d bin src/*.java
java -Dfile.encoding=UTF-8 -cp bin Lab1 data/"Easy Test.txt"
```

### 3.3 观察

- 图中每条边的权重等于两个词连续出现的次数。  
- 桥接词提示与生成新文本在不存在路径时给出友好提示，不影响主流程。  
- PageRank 迭代 100 次后收敛，被孤立节点贡献通过平均分发。  
- 随机游走记录到本地文件，方便人工比对。

## 四、Git 实验总结（根据实验手册）

1. 在 `软件工程lab/Lab1/` 初始化 Git 仓库，将 `src/`、`data/`、`docs/` 等纳入版本控制，并多次提交模拟实验流程。  
2. 修改后用 `git status`/`git diff` 查看，再次提交、撤销最近一次提交以熟悉 `git reset --soft` 或 `git revert`。  
3. 在 GitHub 上创建 `Lab1-学号` 仓库并添加远程，完成推送。  
4. 管理分支：在 `master` 上创建 `B1`、`B2`，在 `B2` 上创建 `C4`，在各分支上对 `Lab1.java` 等文件做不同修改并提交，再将 `C4` 合并回 `B1`、处理冲突。  
5. 将 `B2` 的修改合并到以学号命名的新分支并推送，在 GitHub 页面上查看变迁；删除已合并分支。  
6. 在 IDE（如 IntelliJ 或 VSCode）中启用 Git 集成，提交并推送与命令行操作一致的改动。

## 五、改进与反思

1. 若引入大模型（如 DeepSeek/KiMi）可在 `generateNewText` 里使用模型生成桥接词推荐、以增强新文本的自然度。  
2. 可将图展示扩展为将节点位置与连线导出为 `DOT`/`Graphviz`，方便可视化。  
3. PageRank 可用 TF-IDF 预处理词的初始权重，进一步区分关键词。  
4. 建议将随机游走过程封装为可配置步数，还可以输出访问边的权重。

## 六、运行说明

1. 使用 `javac` 编译所有 `.java` 文件，输出至 `bin/`。  
2. 运行 `Lab1` 时将数据文件路径作为第一个参数，或在交互式提示中输入完整路径。  
3. CLI 会在菜单中完成所有功能，文件输出仅发生在随机游走步骤，结果保存在工作目录的 `random_walk.txt`。
